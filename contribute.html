Become a Contributor

02 Apr 2014, written by Charlie Greenbacker
Tweet

The list of open source software on our Catalog of Tools page will always be incomplete. We plan to add more & more tools over time, but there's no way we will ever cover everything. However, to make the list as comprehensive as possible, we need your help! Have you used an OSS tool in your work as a data scientist, or are you working to develop such a tool? If so, tell the world about it here.

The OSS4DS.com website is built using Jekyll & Bootstrap, and it's hosted on GitHub Pages, making it incredibly easy for anyone to contribute new content. Basically, all you have to do to add new tools to our Catalog is write a little bit of Markdown code and submit a pull request via GitHub. Follow the steps below to set up a local development environment of your own to work on the website.

Getting Started: Fork the Code!

  1. Register for a GitHub account (if you don't already have one)
  2. Visit the OSS4DS repository on GitHub: https://github.com/altamiracorp/OSS4DS
  3. Click on the Fork button to create your own copy of the OSS4DS repository in your GitHub account

Running the Site Locally

  1. Install Ruby (you'll need at least Ruby 1.9.3)
  2. Install bundler with the command: gem install bundler
  3. Install Git (if it's not already installed)
  4. Clone your fork of the OSS4DS code: git clone https://github.com/YOUR-GITHUB-USERNAME/OSS4DS.git
  5. Move into the newly-created OSS4DS directory, e.g., cd OSS4DS/
  6. Install the required Ruby gems for Jekyll, etc. using bundler: bundle install
  7. Install Pygments for source code highlighting
  8. Use Jekyll to compile and serve the website locally: bundle exec jekyll serve --watch
    (the --watch flag tells Jekyll to watch for changes and re-generate the site automatically)

Please note: It's import to use bundle exec when running any of the Jekyll commands to ensure the correct versions of all Ruby gems are used. Check the Jekyll documentation if you get stuck -- it's quite good.

At this point, you should be able to open up your web browser and go to http://0.0.0.0:4000 to view your local copy of the OSS4DS.com website running on your local machine. Here, you can preview any of the additions or changes you've made and refine them until they're ready to be contributed back to the public site.

Adding Tools to the Catalog

Our Catalog of Tools page is actually a blog powered by Jekyll. Adding new tools to the list is as easy as creating a new .markdown file in the _posts directory. You can work from the example files already in that directory, or you can use this handy-dandy template:

---
layout: post
title: "NAME OF TOOL"
author: YOUR NAME
categories: we'll-handle-this or-you-can-look-at-examples
---
Here, you'll write a brief introduction describing the tool and what it's used for. 2-3 sentences is sufficient.

{% raw %}[<img style="float: right" src="{{ site.url }}/img/PATH_TO_LOGO.png" />](http://PROJECT-WEBSITE.io/){% endraw %}

__Creator:__ [NAME OF CREATOR](http://CREATOR-WEBSITE.io/)

__License:__ [OPEN SOURCE LICENSE](link to appropriate opensource.org license page)

__Website:__ [PROJECT-WEBSITE.io](http://PROJECT-WEBSITE.io/)

__Source:__ [github.com/CREATOR/PROJECT](https://github.com/CREATOR/PROJECT)

__Features:__

* List a few interesting features here
* You'll want to include at least two
* But three or four is better

For help with Markdown syntax, go here: Daring Fireball: Markdown Syntax Documentation

It's always best to include an image of the project logo in each post, so whenever possible, be sure to upload an appropriately-sized image file to the img directory and set the img path in the markdown code just below the tool description.

Once your new .markdown file is saved in your _posts directory, you should be able to view it locally by going to http://0.0.0.0:4000/blog/ and clicking on the name of your new tool in the big list.

Submitting a Pull Request

After you've verified your changes locally, submitting your contribution requires two simple steps. First, you'll commit your changes and push them to your fork of the OSS4DS code repository:

git add .
git commit -m "useful message describing changes"
git push origin gh-pages

Please note: if you get a "Page build failure" notification from GitHub at this point, that's ok -- just ignore it!

Next, you'll create a pull request to have your change merged into the main OSS4DS.com website:

  1. Visit your fork on GitHub: https://github.com/YOUR-GITHUB-USERNAME/OSS4DS
  2. Click on the green "Compare & Review" button to go to: https://github.com/YOUR-GITHUB-USERNAME/OSS4DS/compare
  3. Now, click on the green "Create Pull Request" button
  4. Finally, follow the on-screen instructions and click on "Send Pull Request"

Detailed instructions for creating pull requests on GitHub can be found here: Creating a pull request

That's It!

Your part is done. Now it's up to the OSS4DS.com team to review your contribution, make any modifications as appropriate, and merge your pull request into the main website.

And there's no need to stop with adding new tools to the Catalog -- if you've got an idea for how to spruce up the site design, or a new static page you'd like to add (like this one), we'll consider pull requests of that sort as well!
